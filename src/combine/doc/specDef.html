Species Definition File Format

Definition of records begining with the following characters
  • '/' or '!' comment record
  • '#start YYYYDDD HHMMSS' defines starting date and time
  • '#end YYYYDDD HHMMSS' defines ending date and time
  • '#layer lay' defines which layer to process, default is all layers
  • blank records are ignored
  • All other records are read as species definition records
Species definition (CSV) record fields
  1. species name (maximun of 16 characters)
  2. units (maximum of 10 characters)
  3. formular expression (maximun of 512 characters)
Formular expressions supports operators +-*/ and are evaluated from left to right using precedence order of */+-. Order of evaluation can be forced by use of parentheses. When part of an experssion is enclosed in parentheses, that part is evaluated first. Species from input files are defined by their name followed by it's file number enclosed in brackets. Previously defined output species can be defined by using file number zero. Expressions can be continued on addition records by ending the record with a '\' character.
  • Note: This version will accept species definition files in old formats, but they may not be supported in future versions.

Example Species Definition file


/
#start   2001001  000000
#end     2001031  230000
#layer   1  
/
/new species    ,units     ,expression

NO              ,ppbV      ,NO[1]

O3              ,ppbV      ,O3[1]

HNO3            ,ppbV      ,HNO3_UGM3[1] / (2.1724*AIR_DENS[1])

RT              ,cm        ,RT[2]

sqrtRT          ,cm        ,SQRT( RT[2] )

absRT           ,cm        ,ABS( RT[2] )

logRT           ,cm        ,LOG( RT[2]+1.0 )

RT2             ,cm        ,(sqrtRT[0] * sqrtRT[0])
